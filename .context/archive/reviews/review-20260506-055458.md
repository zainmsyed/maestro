## Findings

### Finding 1
- **Severity:** medium
- **Category:** test-gap
- **Summary:** Repository CRUD tests only cover happy paths; negative paths and edge cases are untested.
- **Evidence:** `repository_test.go` tests Create → GetByID → List for all four entities, but never tests GetByID with a missing ID (should return `sql.ErrNoRows`), duplicate primary-key inserts (should fail), or `IsSynthetic` bool round-tripping through INTEGER storage.
- **Recommendation:** Add at least one negative-path test per repository: `GetByID` for a non-existent record, and a bool-round-trip assertion on `epic.IsSynthetic`.
- **Rule candidate:** Repository tests should verify both success paths and at least one error path per public method.

### Finding 2
- **Severity:** low
- **Category:** simplification
- **Summary:** `scanner` interface is defined in `epic.go` but used by all four scan helpers across the package.
- **Evidence:** `type scanner interface{ Scan(...any) error }` is declared at line 78 of `internal/repository/epic.go` and imported/used implicitly by `scanFeature`, `scanSprint`, and `scanAudit` in other files.
- **Recommendation:** Move the `scanner` interface to `helpers.go` where the other shared repository utilities live.
- **Rule candidate:** Package-local interfaces consumed by multiple files should live in the package’s shared utility file, not alongside a single consumer.

### Finding 3
- **Severity:** low
- **Category:** simplification
- **Summary:** `Migrate` splits the embedded schema by semicolon, which is fragile if future statements contain `;` inside string literals or comments.
- **Evidence:** `internal/db/db.go` line 60: `for _, statement := range strings.Split(schemaSQL, ";") { ... }`. This works for the current schema but will silently break if a `CHECK` constraint or `DEFAULT` value ever contains a semicolon.
- **Recommendation:** Either split on `;\n` after trimming whitespace, or switch to a lightweight migration library (e.g., `golang-migrate/migrate`) before the schema stabilizes.
