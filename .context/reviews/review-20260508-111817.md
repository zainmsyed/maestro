## Findings

### Finding 1
- **Severity:** low  
- **Category:** dead-code  
- **Summary:** The `rowIndex` variable in `{#each group.rows as row, rowIndex}` is declared but never used. It is a leftover from an earlier iteration where the previous row was passed to `shouldShowRow`.  
- **Evidence:** `DataTable.svelte` line 134: `{#each group.rows as row, rowIndex}`. The `rowIndex` is not referenced anywhere in the loop body.  
- **Recommendation:** Remove `, rowIndex` from the `{#each}` declaration.  
- **Rule candidate:** Unused index variables in `{#each}` blocks should be removed to keep templates clean.

### Finding 2
- **Severity:** low  
- **Category:** simplification  
- **Summary:** The story-row expand-spacer uses an inline `style` attribute (`style="width: 18px; flex-shrink: 0;"`) instead of a CSS class. This duplicates layout rules that should live in the `<style>` block alongside the `.expand-spacer` class definition.  
- **Evidence:** `DataTable.svelte` line 162. The `.expand-spacer` class already exists in `<style>` but only sets `width: 18px` — the inline style adds `flex-shrink: 0`.  
- **Recommendation:** Move `flex-shrink: 0` into the `.expand-spacer` class definition and remove the inline `style` attribute.  
- **Rule candidate:** Inline style attributes should only be used for dynamic values computed at runtime; static layout rules belong in CSS classes.

### Finding 3
- **Severity:** low  
- **Category:** simplification  
- **Summary:** The `group-row` `<td>` uses a hardcoded `colspan={columns.length + 2}` to span across all columns including the two hardcoded ones (Story Points, Date Source). This magic number is fragile — adding or removing a column requires updating this expression in two places (the JSX template and any mental model of the table width).  
- **Evidence:** `DataTable.svelte` line 130: `<td colspan={columns.length + 2}>{group.label}</td>`. The `+ 2` accounts for Story Points and Date Source which are outside the `columns` array.  
- **Recommendation:** Either (a) add Story Points and Date Source as entries in the `columns` array with a flag indicating they are non-sortable, or (b) define a `totalColumns` derived value (e.g., `$: totalColumns = columns.length + 2`) and use it for both `colspan` and any width calculations.  
- **Rule candidate:** Magic numbers that encode implicit structural relationships should be replaced with named constants or derived values.
