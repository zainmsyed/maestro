## Findings

### Finding 1
- **Severity:** medium
- **Category:** bug
- **Summary:** Today and sprint boundary overlays use a fixed `gridOffset` and independent date-to-pixel math, so the vertical lines can drift from SVAR's actual timeline when the table pane width changes or the chart is horizontally scrolled.
- **Evidence:** `GanttView.svelte` sets `const gridOffset = 440` and passes it to both overlays. `TodayLine.svelte` and `SprintBoundaryOverlay.svelte` compute `left = gridOffset + daysBetween(...) * cellWidth`, but SVAR owns chart scrolling and grid sizing internally. The grid columns include a flex column (`{ id: 'text', flexgrow: 1 }`), so the left edge of the chart is not guaranteed to remain 440px.
- **Recommendation:** Either use SVAR markers (`markers` prop) for the today line and sprint boundaries, or anchor overlays inside the SVAR chart scroller using measured chart left/scroll offsets from the DOM/API rather than a fixed constant.
- **Rule candidate:** Overlay components that align to third-party virtualized/scrolled content must derive position from the rendered scroller/API, not fixed layout constants.

### Finding 2
- **Severity:** low
- **Category:** simplification
- **Summary:** `taskCss` is exported from `svarBridge.ts` but only used internally, creating an unused-export finding.
- **Evidence:** Fallow reports `frontend/src/lib/svarBridge.ts:86 — unused exports`. The exported `taskCss` function is called by `buildFeatureTask`/`buildEpicTask` inside the same module but is not imported elsewhere.
- **Recommendation:** Remove the `export` modifier unless a future story needs to import the function directly.
- **Rule candidate:** Do not export module-private helpers solely for tests or future use; export only API surface consumed outside the module.

### Finding 3
- **Severity:** low
- **Category:** simplification
- **Summary:** `TodayLine.svelte` and `SprintBoundaryOverlay.svelte` duplicate prop names and date math helpers, causing duplicate-export and duplicate-code findings.
- **Evidence:** Fallow reports duplicate exports on shared props (`timelineStart`, `timelineEnd`, `cellWidth`, `gridOffset`) and duplicate code between `frontend/src/components/SprintBoundaryOverlay.svelte` and `frontend/src/components/TodayLine.svelte`. Both components define `dayMs`, `startOfDay`, and `daysBetween`.
- **Recommendation:** Extract shared date-to-pixel helpers and shared overlay prop typing into a small `frontend/src/lib/ganttOverlay.ts` module, or consolidate both overlays into a single generic timeline marker component.
- **Rule candidate:** When two Svelte components share positioning math, extract the math into a plain TypeScript helper before adding a second copy.

### Finding 4
- **Severity:** low
- **Category:** simplification
- **Summary:** `styleForVariant` exceeds the complexity threshold and would be easier to maintain as a lookup table.
- **Evidence:** Fallow reports `frontend/src/screens/GanttView.svelte:77 — function styleForVariant exceeds complexity threshold (score: 5)`. The function is an if-chain mapping `is_synthetic` / `status_variant` to CSS strings.
- **Recommendation:** Replace the conditional chain with a `Record<SvarStatusVariant | 'synthetic', string>` lookup and a single key selection.
- **Rule candidate:** Replace status/style conditional chains with lookup tables when mappings are static.
